SMB & IPC & RPC ​

1. SMB (Server Message Block) 445 ​

SMB 是一种网络文件共享协议，允许应用程序读取、写入远程服务器上的文件，以及进行远程服务器资源的通信（如打印机）。在你的数据包中，SMB2 协议被多次提及，表明正在进行文件共享、目录枚举等操作。

  • SMB2：是 SMB 协议的更新版本，较 SMB1 更高效，主要用于现代 Windows 系统。
  • 常见操作：
    • Tree Connect：请求建立对共享资源（如文件夹）的连接。
    • Create Response File：表示创建文件时的响应。
    • EnumDomains：枚举域信息，可能在进行域控制器或其他网络信息收集时使用。

2. DCERPC (Distributed Component Object Model RPC) 135 ​

DCERPC 是一种协议，用于在分布式计算机系统上进行远程过程调用。它基于 RPC（Remote Procedure Call）协议，使得应用程序能够调用另一台计算机上的方法。DCERPC 是用于访问系统服务的关键协议，尤其在 Windows 系统中用于很多管理和远程执行功能。

  • 常见操作：
    • Read Request 和 Response：表示尝试读取某些文件或资源。
    • QueryDisplayInfo：查询显示信息，通常用于远程获取系统的用户信息或状态。
    • STATUS_MORE_ENTRIES：表明返回的查询结果分页，还需要更多的数据包来完成请求。

3. IPC (Inter-Process Communication) ​

IPC 是一种机制，允许在同一计算机或不同计算机上的进程之间进行通信。在 Windows 系统中，$IPC 共享用于系统进程间的通信，而 IPC$ 共享通常不会显示在常规的网络共享列表中，它是隐式的。

  • $IPC 用于远程过程调用（RPC），通过 \\ip\$IPC 共享来进行进程间通信。这个共享并不存储普通文件，而是用于系统之间的内部通信（如管理任务、服务交互等）。

它们如何协同工作？ ​

  • SMB 和 IPC：在网络上，SMB 用于文件和资源共享，而 $IPC 共享用于处理远程管理和进程间通信。通过 $IPC，Windows 系统可以让远程计算机执行特定任务（如调用远程过程）。

  • SMB 和 DCERPC：当系统执行远程管理任务时，可能会通过 SMB 协议建立连接，再使用 DCERPC 进行远程过程调用。例如，通过 DCERPC 查询或更改系统配置时，SMB 负责建立连接。

  • DCERPC 和 IPC：DCERPC 也可以通过 IPC 管道传输数据。通过 $IPC，你可以执行远程命令或查询系统状态，而 DCERPC 提供了通过网络进行调用的机制。

渗透测试中的应用： ​

在渗透测试中，这些协议的流量常常是攻击者尝试获取远程机器信息、提升权限、执行命令或绕过防护的目标。

  1. SMB：可以通过暴力破解 SMB 登录凭据来进行渗透，或者利用 SMB 漏洞（如 EternalBlue）进行远程代码执行。
  2. DCERPC：可以用于通过远程过程调用执行命令、获取系统信息，甚至进行横向移动（例如利用已知的漏洞如 MS08-067）。
  3. IPC：通过 $IPC，攻击者可以与远程计算机的进程进行通信，可能执行远程管理任务或者绕过某些安全控制。
