$substitutions = array(
		'&'  => '',

		'$'  => '',

Vulnerability: Cross Site Request Forgery (CSRF) ​

Security Level: low ​

按照迪总的思路来：

SSRF & CSRF

  1. 我们使用普通的用户登录，1337用户，修改密码抓包尝查看

  2. 构建修改密码的链接，修改密码为123的payload

    php
    # 最基础的，这种攻击显得有些拙劣，链接一眼就能看出来是改密码的，而且受害者点了链接之后看到这个页面就会知道自己的密码被篡改了
http://192.168.75.128:8006/vulnerabilities/csrf/?password_new=123&password_conf=123&Change=Change

    一般是要用短链接来隐藏url，不过ip地址作为域名是无法生成短链接的

  3. 第三步就是让登陆着管理员用户的人来点这个链接（社会工程学）

    现在模仿管理员点击这个链接，此时我登陆着管理员用户，点击后密码就被更改了

    还有别的方法，就是构建一个网页，网页里面是XSS代码，点击后构建的网页链接后自动加载需改密码的payload，这种方法比上面的隐蔽
